# Astrology-Transcribe-Mac (占星學語音轉錄與 AI 校正工具)

這是一個專為 Mac (Apple Silicon，如 M1/M2/M3) 設計的本地端語音轉錄與校正工具。
系統結合了 `mlx-whisper` 的極速語音辨識，以及透過本地 `Ollama` 運行的本地大型語言模型 (如 Qwen 2.5)，將語音檔案轉為高準確度的逐字稿與字幕檔，特別針對占星學的專有名詞進行了優化與防幻覺處理。

## 系統需求
- **硬體**：配備 Apple Silicon 晶片的 Mac (M1/M2/M3)
- **作業系統**：macOS
- **Python 版本**：Python 3.9 或以上
- **本地 LLM 引擎**：已安裝並執行中的 [Ollama](https://ollama.com/)，且需事先下載指定的模型：
  ```bash
  ollama pull gemma3:12b
  ```

## 安裝教學
1. 準備好上述環境，並確認 Python 已正確安裝 (`python --version`)。
2. 安裝本專案所需的 Python 依賴套件：
   ```bash
   pip install -r requirements.txt
   ```

## 目錄結構說明
此專案會依賴幾個特定的資料夾來處理檔案，若無則程式執行時會自動建立：
- `audio/`：來源資料夾。請將你要處理的語音/影片檔 (`.aac`, `.mp3`, `.m4a`, `.mp4`, `.wav`) 放入這裡。
系統會自動將 `.aac`, `.mp3`, `.m4a` 格式轉為 16kHz 單聲道 `.wav` 檔進行高精度轉錄，轉錄完成後自動刪除暫存檔。
- `output/`：初剪資料夾。`transcribe_engine.py` 產出的初步轉錄字幕 (`.srt`) 會存放在這。
- `final_output/`：完稿資料夾。經過 AI 校正後的最終成品 (`.srt`, `.md` 格式短文) 會存放在這。
- `models/`：當第一次執行轉錄時，會自動下載並儲存 `mlx-whisper` 使用的 Whisper 模型。

## 執行流程與使用方式

本工具分為「轉錄」與「校正」兩大步驟。

### 第一步：語音極速轉錄 (Whisper)
將您的檔案放入 `audio/` 資料夾後，執行：
```bash
python transcribe_engine.py
```
- 初次執行時，程式會自動花一些時間從 Hugging Face 下載 `mlx-community/whisper-large-v3-mlx` 模型。
- 轉錄過程會自動過濾常見的 Whisper 產生的幻覺詞彙（如「訂閱」、「打賞」等）。
- 完成後，請至 `output/` 資料夾檢視產出的 `.srt` 檔案。

### 第二步：AI 智能校正與文章生成 (Ollama)
轉錄完畢的檔案可能缺乏標點符號，或是有些許同音異字。我們透過本地的 Ollama AI 幫忙校正錯字並補上標點符號，同時會把它排版成適合閱讀的 Markdown 文章。

執行以下腳本進行批次校正：
```bash
# 預設使用 qwen2.5:7b 進行校正
# 已優化為使用 gemma3:12b 進行專業校正與排版
python sub_edit-14b.py

### 第三步：專業助教重點整理 (sub_summy.py)
如果您需要將幾萬字的逐字稿濃縮成精華課程筆記，可以使用此腳本：
```bash
python sub_summy.py
```
- **專業人設**：AI 以「占星課程助教」身分進行整理。
- **美化排版**：自動使用 GitHub Alert (Tip, Note, Important) 語法標註核心知識。
- **防遺漏邏輯**：強制要求輸出字數需達原始稿件的 30% 以上，確保細節不流失。
```
- `.md` 檔案頂部會自動加上 YAML 標籤記錄建立時間與占星相關的 Tags。
- 校正過程中會保持原汁原味的口語表達，不會過度修改句子或替換詞彙。
- 產出的完稿 `.md` 與 `.srt` 會儲存在 `final_output/` 資料夾中。

## 專有名詞自訂 (astrology_data.py)
如果您有特定常被辨識錯誤的專有名詞，或是想增加新的詞彙（例如特定的星座名、宮位、相位名稱），您可以直接打開 `astrology_data.py` 修改 `glossary` 陣列。
這份名單會作為 Whisper 轉錄與 AI 校正時的重要參考依據。

## 0310 增加重點筆記的功能
- 自動切段與合併：現在腳本在讀取內容時，會先以 2000 字為單位切割成數個小段落。
- 強迫長度輸出：輸出的筆記字數不得少於收到的逐字稿內容字數的 30%，保證細節不會被省略。

## 新增 FFmpeg 音訊前置處理
- 若來源格式為 `.aac`、`.mp3` 或 `.m4a`，系統會自動在背景呼叫 FFmpeg 建立 `16kHz`, 單聲道 (`mono`) 的暫存 `.wav` 檔，這麼做可以大幅提升 Whisper 的辨識準確度。
- 轉錄完畢後會將生成的暫存 wav 檔自動刪除，並且輸出時依然會保留原始好的字幕檔名稱。

## 0312 深度優化升級 (Gemma 3)
- **模型遷移**：全面升級至 Google 的 `gemma3:12b` 模型，針對長文本理解力與專業詞彙對齊。
- **助教語氣**：AI 人設調整為「占星助教」，回應語氣溫暖、專業且條理分明。
- **GitHub 風格筆記**：在 Markdown 中加入現代化標註元素，讓複習更有效率。
- **程式邏輯加強**：修復了分段處理時的內存標記清空問題，確保長時長音檔也能一氣呵成產出筆記。
