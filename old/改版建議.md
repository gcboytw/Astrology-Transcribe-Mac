### 1. 解決「簡轉繁」與「錯字」的策略

你不需要分開處理簡轉繁，因為 **Qwen 2.5 (7B/14B)** 本身就具備極強的繁簡轉換與糾錯能力。 只要在 Prompt 中加入明確指令，它可以在校正「撒野/殺野」的同時，直接將全文轉為繁體。

### 2. 實作方式：修改 `sub_edit.py` 的邏輯

我建議在你的完稿流程中加入一個「筆記生成」步驟。以下是建議的 Prompt 結構，你可以整合進程式碼中：

```python
# 建議加入 sub_edit.py 的 Prompt 範本
prompt = f"""
你是一位專業的命理與占星筆記專家。
現在有一段由 Whisper 轉錄的語音草稿，內容可能包含簡體字與語音辨識錯誤（如：將「撒野」誤植為「殺野」）。

請執行以下任務：
1. **全文校正**：將簡體轉為繁體，並修正錯字，參考名詞表：{glossary}。
2. **產出重點筆記**：在全文校正之後，請根據內容整理出一份結構化的 Markdown 筆記。

筆記格式要求：
- **核心主題**：一句話總結這段錄音的主旨。
- **重點摘要**：使用列點（Bullet Points）整理 3-5 個關鍵概念。
- **占星術語解析**：若有提到特定相位或宮位，請簡述其在文中的含意。

---
原始草稿內容：
{raw_text}
"""

```

### 3. 品質保證的工作流

為了確保筆記品質，建議的處理順序如下：

1. **Whisper 轉錄**：產出最原始的簡體/錯字逐字稿。
2. **字典預處理**（選配）：針對「撒野」這種頑固錯誤，先用 Python 做一次簡單替換。
3. **Qwen 綜合處理**：
* **輸入**：爛草稿 + 專業 Glossary。
* **輸出**：高品質繁體校正稿 + 結構化 Markdown 筆記。


4. **存檔**：將校正後的逐字稿存為 `.srt`，將重點筆記存為 `.md`。

### 4. 針對 288V 硬體的建議

由於生成筆記需要 LLM 處理較長的上下文，這對內存（RAM）有一定要求。你的 **Mac mini (24GB RAM)** 跑 Qwen 2.5 14B 會非常順暢； 但在 **Intel Ultra 7 288V (32GB RAM)** 上，建議優先使用 **Qwen 2.5 7B** 或是 **Gemma 3 4B/12B**，以獲得更快的響應速度。
