既然你無法將所有的誤判詞一一列入 `glossary`，我們可以透過**改變 LLM 的校正邏輯**來提升準確性。目前你的 `glossary` 雖然標註了「常被誤聽為」，但對 AI 來說那只是括號內的補充資訊，它不一定知道該如何「強制執行」。

針對「撒野」被誤判為「殺野」這類非占星專有名詞、但屬於生活常識的錯誤，建議採取以下三種策略：

### 1. 修改 `sub_edit.py` 的 Prompt 策略 (System Prompt)

目前的校正可能太過依賴模型對占星的理解，你需要明確告訴模型：**「優先修復語音辨識常見的同音異義錯誤」**。

在呼叫 Ollama 的 Prompt 中加入以下指令：

> 「你現在是一位具備深厚占星背景與文學素養的編輯。在校正時請遵循：
> 1. **同音異義檢查**：語音辨識常會產生錯字（例如：將『撒野』辨識為『殺野』、『合相』辨識為『河相』）。請根據上下文語境，將這些不符合常理或占星邏輯的詞彙修正。
> 2. **參考括號備註**：若 `glossary` 中有標註『常被誤聽為...』，請務必以此作為首要修正依據。」
> 
> 

---

### 2. 結構化你的 `glossary` (資料驅動校正)

與其在 `glossary` 裡寫長句子，不如將它拆解成 LLM 更好吸收的格式。建議將 `astrology_data.py` 改為**字典 (Dictionary)** 結構：

```python
# astrology_data.py 優化版
# 占星專有名詞
astrology_terms = ["相位", "白羊座", "第十宮", "辛納斯特里"]

# 常見語音誤判對照表 (包含生活用語)
common_corrections = {
    "公衛": "宮位",
    "工位": "宮位",
    "精心": "金星",
    "驚心": "金星",
    "交匯": "交會",
    "食運": "時運",
    "殺野": "撒野", # 雖然不是占星詞，但常錯就放進來
    "河相": "合相"
}

```

然後在 `sub_edit.py` 讀取這張表，並在 Prompt 中告訴 AI：**「如果看到左邊的詞，請根據語境判斷是否應替換為右邊的詞。」**

### 3. 使用語境引導 (Contextual Healing)

「撒野」這類詞之所以會錯，是因為 Whisper 在辨識時失去了情感語境。你可以嘗試在 `transcribe_engine.py` 的 `initial_prompt` 加入一些**語氣助詞或生活化例句**。

* **目前的做法**：只放名詞。
* **更好的做法**：放一段包含這些詞的自然段落。
> `initial_prompt` = "這是一段占星講座，老師會提到金星、相位與宮位。有時候會開玩笑說學生不要撒野，要觀察時運的交會。"



---

### 4. 終極方案：兩階段校正 (Two-Pass Correction)

如果你追求極高的準確度，可以讓 Qwen 做兩次處理：

1. **第一遍 (詞彙修復)**：只對照 `common_corrections` 字典做精準替換。
2. **第二遍 (語意潤色)**：再進行整體的標點符號與占星專業度調整。
