## 📑 學習筆記 [00:00:00 - 00:04:08]：現代 Python 專案管理概覽

> [!TIP]
> **本段核心**：本段影片介紹了現代 Python 專案管理的兩個主要流派：Conda 和官方 Python 生態系統，並說明瞭 Conda 的優點和官方生態系統中 pip、uv 和 poetry 等工具的重要性，以及虛擬環境的必要性。

### 💡 核心知識點
* **Python 專案管理歷史**：早期 Python 專案管理較為隨意，但官方 PEP 標準持續改善。
* **Conda 系統**：
    * *補充*：Conda 是一個跨語言的開發平臺，支援 Python、Go、Rust、C++、R 等多種語言，並提供多語言支援、依賴管理和虛擬環境等功能。它有自己的設定檔、套件儲存庫，甚至會自行編譯 Python 直譯器。
* **官方 Python 生態系統**：包含 Setuptools、Hatchling、pip、uv 和 poetry 等工具。
* **虛擬環境**：
    * *補充*：虛擬環境用於為每個專案建立隔離和乾淨的 Python 程式碼環境，避免版本衝突和相依地獄。使用 `python -m venv .venv` 建立虛擬環境。`.venv` 只是虛擬環境的名稱，建議使用此命名方式，方便 IDE 識別。

### 🔍 具體範例或案例
* **"hello world" 專案**：
    * 專案目錄中只有一個 `main.py` 檔案，使用 Flask 框架實現簡單的 "hello world" 應用程式。
    * 初始狀態下，程式碼會因為缺少 Flask 函式庫而報錯。
    * 使用 `pip install flask` 安裝 Flask 後，程式可以正確執行，但會造成全域性的版本衝突風險。

### 🛠️ 實作建議與行動方針
* **避免全域性安裝**：不要直接使用 `pip install` 全域性安裝函式庫，因為容易造成版本衝突。
* **使用虛擬環境**：為每個專案建立虛擬環境，使用 `python -m venv .venv` 指令。
* **關注官方工具**：影片結尾會介紹 uv 和 poetry 等最佳實踐工具，建議持續關注。

## 📑 學習筆記 [00:04:45 - 00:08:57]：Python 虛擬環境與套件管理

> [!TIP]
> **本段核心**：本段講解了 Python 虛擬環境的原理、使用方式，以及如何透過 `requirements.txt` 和 `pyproject.toml` 檔案來管理專案的套件依賴，並說明瞭 `pyproject.toml` 的優勢。

### 💡 核心知識點
* **虛擬環境的作用**：
    * 避免套件衝突：每個專案都可以擁有自己的獨立虛擬目錄，包含專案所需的特定套件。
    * `sys.path` 的修改：虛擬環境啟動後，會修改 Python 的 `sys.path` 變數，將虛擬目錄加入搜尋路徑，優先載入虛擬環境中的套件。
* **`requirements.txt` 的使用**：
    * 記錄已安裝套件：使用 `pip freeze` 指令將目前虛擬環境中所有套件及其版本號碼輸出到 `requirements.txt` 檔案。
    * 方便分享：他人可透過 `pip install -r requirements.txt` 指令安裝專案所需的所有套件。
    * 缺點：無法區分直接和間接相依套件，容易產生不必要的孤立依賴。
* **`pyproject.toml` 的優勢**：
    * 統一配置檔案：取代多個分散的設定檔（例如 `mypy.ini`, `pytest.ini`），簡化專案根目錄。
    * 宣告直接依賴：僅在 `dependencies` 列表中宣告專案直接依賴的套件，避免產生孤立的依賴。
    * 方便移除依賴：移除套件時，只需刪除 `pyproject.toml` 中的對應行。

### 🔍 具體範例或案例
* **安裝 Flask 到虛擬環境**：
    * 啟用虛擬環境後，使用 `pip install Flask` 將 Flask 安裝到虛擬環境中。
* **產生 `requirements.txt` 檔案**：
    * 使用 `pip freeze > requirements.txt` 將目前虛擬環境中的所有套件及其版本號碼輸出到 `requirements.txt` 檔案。
* **安裝 `requirements.txt` 中的套件**：
    * 使用 `pip install -r requirements.txt` 指令安裝 `requirements.txt` 檔案中列出的所有相依套件。
* **使用 `pyproject.toml` 檔案**：
    * 在 `pyproject.toml` 檔案中新增 `dependencies` 列表，將 Flask 直接依賴複製到列表中。
    * 刪除 `requirements.txt` 檔案。
    * 使用 `pip install .` 指令安裝專案的依賴。

### 🛠️ 實作建議與行動方針
* **善用虛擬環境**：在每個 Python 專案中使用虛擬環境，避免套件衝突。
* **優先使用 `pyproject.toml`**：盡量使用 `pyproject.toml` 檔案來管理專案的套件依賴，取代 `requirements.txt`。
* **只宣告直接依賴**：在 `pyproject.toml` 的 `dependencies` 列表中，只宣告專案直接依賴的套件，避免產生孤立的依賴。

## 📑 學習筆記 [00:09:38 - 00:13:53]：專案相依性管理與工具簡介

> [!TIP]
> **本段核心**：本段講解了使用 `venv` 解決專案相依性問題的過程，並介紹了 Poetry、uv 和 PDM 等第三方工具，它們能簡化環境管理和相依性安裝的流程，提升開發效率。

### 💡 核心知識點
* **`venv` 與 `pip install -e .` 的使用**：
    * 使用 `python -m venv .venv` 建立虛擬環境，將專案安裝到虛擬環境中。
    * `pip` 安裝時會執行建置（打包專案）和安裝兩個步驟。
    * 初始安裝時，`pip` 會將原始程式碼複製到 `site-packages` 目錄，導致程式碼副本增加，變更無法自動同步。
    * 使用 `pip install -e .` 安裝時，不複製原始程式碼，而是建立連結檔案，確保程式碼變更立即生效。
* **`pyproject.toml` 的限制**：
    * 使用 `venv` 和 `pyproject.toml` 管理相依性時，無法像 `pip install` 一樣直接新增相依性。
    * 需要手動查詢套件名稱和版本，並寫入配置檔案，流程繁瑣且容易出錯。
* **第三方工具簡介 (Poetry, uv, PDM)**：
    * 這些工具是 `venv` 和 `pip` 的高階封裝，提供更簡潔的介面。
    * 像是 Poetry、uv 和 PDM 等工具，能自動化環境管理和相依性安裝。

### 🔍 具體範例或案例
* **使用 `uv` 管理專案的流程**：
    1.  **建立虛擬環境**：`python -m venv .venv`
    2.  **啟用虛擬環境**：`source` 指令
    3.  **新增相依性**：`uv add flask` (自動修改 `pyproject.toml`，建立虛擬環境，並安裝相依套件)
    4.  **同步專案**：`uv sync` (讀取 `pyproject.toml`，設定虛擬環境，並安裝所有相依套件)
    5.  **執行程式碼**：`python main.py` 或 `uv run main.py` (後者自動在虛擬環境中執行指令)

### 🛠️ 實作建議與行動方針
* **善用第三方工具**：如果開發流程中需要頻繁新增或更新相依性，建議使用 Poetry、uv 或 PDM 等工具來簡化流程。
* **熟悉 `uv` 指令**：`uv add`、`uv sync` 和 `uv run` 指令能有效提升開發效率。
* **觀看相關影片**：若對 `uv` 工具感興趣，可參考說明欄位中的連結，深入瞭解其更多用法。
