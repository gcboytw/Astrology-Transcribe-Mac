### [00:02:48]
假設我們有一個專案，在目錄中，只有一個程式碼檔案，main.py。它的功能非常簡單：它只是用 Flask 寫的一個 "hello world"。在一個新的環境中，這個函式庫通常沒有安裝，所以直接執行會導致錯誤。這很容易解決。 我們的第一反應是執行 `pip install flask` 來安裝它。安裝後，程式確實可以正確執行。但有一個重要的細節需要注意。當你這樣安裝 Flask 時，這個環境由電腦上所有 Python 專案共享。這種共享機制，在實際開發中，可能會導致兩個非常棘手的問題。第一個是版本衝突。

### [00:03:30]
例如，我們的新專案安裝了 Flask 3.1.1 版本，但電腦上另一個舊專案可能只與 3.0 版本相容。如果我們為了新專案全域性升級 Flask 函式庫，舊專案可能會因為版本不相符而停止運作。第二個是複雜的相依關係，我們通常稱之為「相依地獄」。一個函式庫通常會依賴於幾個其他的函式庫，而那些函式庫又各自有自己的相依關係。這種層層巢狀，可能會導致更多版本衝突。虛擬環境就是為瞭解決這個問題而設計的。它可以為每個專案建立一個隔離的和乾淨的 Python 程式碼環境。我們可以透過 `python -m venv .venv` 建立一個虛擬環境。
